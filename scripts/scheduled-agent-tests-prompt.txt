## HIGH SEVERITY

- [ ] B01: Extra Attack does not auto-chain — returns to > prompt between strikes
  Sources: solo-fighter BUG-3, boss-fight BUG-2, wounded-fighter BUG-3, solo-monk BUG-2
  Flow: CombatOrchestration
  Description: Level 5 Fighter/Monk Extra Attack requires manual re-input for 2nd attack.
  After first attack resolves, prompt returns to > with no indication a second attack is available.
  Expected (5e 2024): Attack action with Extra Attack should chain all attacks automatically
  or at minimum show "You have 1 attack remaining" before returning to prompt.

- [ ] B02: Second attack parsed as off-hand TWF with Handaxe + Disadvantage instead of Extra Attack
  Sources: wounded-fighter BUG-1, wounded-fighter BUG-2
  Flow: CombatOrchestration
  Description: After first attack, sending "I attack again" triggers Handaxe off-hand TWF
  with disadvantage instead of using the same weapon for Extra Attack. Explicit weapon
  specification ("with my longsword") is also ignored — server still selects Handaxe.
  Expected: Extra Attack uses same weapon by default, honors explicit weapon choice.

- [ ] B03: AI Flurry of Blows declared but never executed (Vex Nightthorn)
  Sources: monk-vs-monk BUG-1
  Flow: AIBehavior, ClassAbilities
  Description: Vex chains "Shadow Fist + Flurry of Blows" in LLM text, but only Shadow Fist
  executes. The Flurry of Blows bonus action is narrated but never resolved mechanically.
  Expected: AI should either execute both action + bonus action, or LLM should be guided
  to declare them separately so each gets resolved.

- [ ] B04: Action Surge attack count message "4 total remaining" is off
  Sources: boss-fight BUG-3, wounded-fighter BUG-4
  Flow: ActionEconomy, CombatOrchestration
  Description: After using one attack then Action Surge, message says "4 total attacks remaining"
  but only 3 more attacks should be available (1 used + 3 remaining = 4 total in round).
  The word "remaining" is misleading — should say "total this round" or correctly count remaining.

- [ ] B05: Server auto-selects wrong weapon when player specifies one explicitly
  Sources: solo-fighter BUG-2
  Flow: CombatOrchestration
  Description: Player says "attack with longsword" at 10ft range, server silently swaps to
  Handaxe (thrown) without explanation. Should either use the specified weapon and show
  range error, or explicitly tell the player why the weapon was changed.

## MEDIUM SEVERITY

- [ ] B06: Off-hand Handaxe attack applies Disadvantage with no apparent reason
  Sources: wounded-fighter BUG-1
  Flow: CombatOrchestration, CombatRules
  Description: Off-hand attacks are prompted with "Disadvantage! Roll 2d20 and take the lower."
  5e 2024 Two-Weapon Fighting does NOT impose disadvantage. It simply omits ability modifier
  from damage (unless TWF style taken). Investigate why disadvantage is applied.

- [ ] B07: Melee range rejection threshold shows 20ft instead of 5ft for longsword
  Sources: boss-fight BUG-1, solo-fighter note, wounded-fighter note
  Flow: CombatRules
  Description: Error message says "Target is out of range (30ft > 20ft)" for longsword.
  Longsword has 5ft reach. The 20ft threshold is wrong — likely weapon data issue.
  Attacks DO succeed at 5ft, so only the validation/error message is affected.

- [ ] B08: Grapple/shove bypasses tabletop dice flow entirely
  Source: plan-grapple-tabletop-dice-flow.prompt.md (Bug 1)
  Flow: CombatOrchestration, CombatRules
  Description: GrappleHandlers calls programmatic ActionService path that auto-rolls with
  seeded RNG. Player never rolls d20 for grapple/shove attempts.
  Approved plan exists at plan-grapple-tabletop-dice-flow.prompt.md — implement it.

- [ ] B09: Grapple/shove missing advantage from target conditions (Stunned, Paralyzed, etc.)
  Source: plan-grapple-tabletop-dice-flow.prompt.md (Bug 2)
  Flow: CombatRules, ReactionSystem
  Description: Attack roll on grapple/shove doesn't consider Stunned/Paralyzed/etc.
  for advantage and doesn't auto-fail STR/DEX saves for incapacitated targets.
  Approved plan exists — implement it.

- [ ] B10: Attack roll result printed twice in CLI output
  Sources: boss-fight BUG-4
  Flow: CombatOrchestration (response assembly)
  Description: Every hit shows the roll result line twice (before and after narrative text).
  E.g., "16 + 6 = 22 vs AC 11. Hit!" appears twice with narrative sandwiched between.
  Cosmetic but confusing.

- [ ] B11: Goblin AI only moves, never attacks on the same turn
  Sources: solo-fighter BUG-1, solo-monk note
  Flow: AIBehavior
  Description: AI monster moves to melee range then ends turn without attacking.
  In D&D 5e, movement and action are independent. Monster should move AND attack.
  Makes combat trivially easy when monsters never deal damage.

- [ ] B12: Stunning Strike on lethal hit — ki spend not visible, CON save skipped
  Sources: solo-monk BUG-1
  Flow: CombatOrchestration (roll state machine)
  Description: When Stunning Strike kills the target, no CON save prompt appears (reasonable
  optimization) but ki point spend is also not confirmed. Unclear if ki was deducted.
  Expected: Ki should be spent regardless of whether damage is lethal (declared before outcome).
  At minimum, response should confirm "Spent 1 ki" even on lethal.
